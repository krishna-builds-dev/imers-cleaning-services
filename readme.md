# IMERS — Premier Commercial Cleaning Services

> A high-performance, fully static marketing website for **IMERS**, a professional commercial cleaning company based in NSW & NT, Australia. Built with a dark-mode design, custom Tailwind CSS design system, and a secure serverless contact form deployed on Vercel.

---

## 📌 What This Project Is

This is the official client-facing website for IMERS Commercial Cleaning. It is a **single-page static site** that showcases the company's services, process, portfolio, and provides a lead-generation contact form.

### Pages / Sections
| Section | Description |
|---|---|
| **Hero** | Full-screen background with CTA — "Request Quote" |
| **Services** | Commercial Cleaning, Office Cleaning, Janitorial Services |
| **Architectural Cleanliness** | Brand differentiator — ISO-certified, HEPA, EPA protocols |
| **About** | 15+ years, 500+ corporate clients |
| **Process** | 4-step workflow: Consult → Assess → Execute → Maintain |
| **Portfolio** | Gallery of completed projects |
| **Contact** | Secure form with email delivery + Cloudflare Turnstile CAPTCHA |

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| **Frontend** | Vanilla HTML5, Tailwind CSS v3 |
| **Fonts** | Manrope (headings), Inter (body), JetBrains Mono (labels) — self-hosted |
| **Images** | Responsive `.avif` format with `srcset` (400w → 1200w) |
| **Contact API** | Vercel Serverless Function (`/api/contact`) |
| **Email** | Nodemailer via SMTP |
| **Bot Protection** | Cloudflare Turnstile CAPTCHA |
| **Analytics** | Cloudflare Web Analytics |
| **Deployment** | Vercel (static site + serverless functions) |
| **CSS Build** | Tailwind CLI (`npm run build`) |

## ⚙️ Local Development

### Prerequisites
- Node.js 18+
- npm

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/krishna-builds-dev/imers-cleaning-services.git
cd imers-cleaning-services

# 2. Install dependencies
npm install

# 3. Copy environment variables
cp .env.example .env
# Then fill in your real values (see Environment Variables section below)

# 4. Start Tailwind in watch mode
npm run dev
```

Open `index.html` directly in your browser, or use the local Express server:

```bash
npm start   # runs server.js on http://localhost:3000
```

> **Note:** The `server.js` Express server is for **local development only**. On Vercel, the site is served as a static site and the contact form uses a serverless function.

## 🚀 Deployment (Vercel)

This project is deployed automatically on every push to `main`.

### How it works on Vercel
1. **Build:** `npm run build` → generates `dist/output.css` from Tailwind
2. **Static files:** `index.html` and all assets are served from the project root (`outputDirectory: "."`)
3. **API:** `api/contact.js` is auto-detected as a Vercel Serverless Function
4. **Framework:** None (`"framework": null`) — plain static site

### Environment Variables (set in Vercel Dashboard)

Go to **Vercel → Project → Settings → Environment Variables** and add:

| Variable | Description |
|---|---|
| `EMAIL_HOST` | SMTP host (e.g. `smtp.gmail.com`) |
| `EMAIL_PORT` | SMTP port (usually `587`) |
| `EMAIL_USER` | SMTP username / email address |
| `EMAIL_PASS` | SMTP password or app password |
| `EMAIL_TO` | Recipient email (where form submissions are sent) |
| `DOMAIN` | Your production URL, e.g. `https://imers.vercel.app/` |
| `TURNSTILE_SECRET_KEY` | Cloudflare Turnstile secret key |

## 📬 Contact Form — How It Works

```
User fills form → Cloudflare Turnstile CAPTCHA validates → 
POST /api/contact → Server validates fields → 
Nodemailer sends email to EMAIL_TO → Success response
```

**Security features:**
- Cloudflare Turnstile CAPTCHA (bot protection)
- Server-side field validation
- HTML escaping on all user input
- CORS locked to production domain (`DOMAIN` env var)
